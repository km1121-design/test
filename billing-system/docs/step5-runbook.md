# Step 5: 検算・本稼働 — 実行手順書（Runbook）

作成日: 2026/07/19
前提: Step 1〜4 完了。計算ルールはすべてユーザー確定済み（消費税四捨五入・
ドライバー税込契約・ネコポス卸¥50・2月試算一致）。

このStepは **GAS のデプロイと実行**（ユーザーの Google アカウント上での作業）が中心。
所要 15〜30 分。

---

## 1. GAS プロジェクト作成（初回のみ）

1. <https://script.google.com> → 新しいプロジェクト（名前例: `配送請求システム`）
2. `billing-system/apps-script/` の4ファイルをそれぞれ同名で貼り付け:
   - `Normalize.gs` / `DataReader.gs` / `Reconcile.gs` / `Billing.gs`
3. プロジェクトの設定（歯車）→ スクリプト プロパティ:

   | プロパティ | 値 | 必須 |
   |---|---|---|
   | `SOURCE_SPREADSHEET_ID` | （既定で実績・経費シートID内蔵。変える場合のみ） | - |
   | `SHIFT_SPREADSHEET_ID` | （既定でシフトシートID内蔵。変える場合のみ） | - |
   | `OUTPUT_SPREADSHEET_ID` | 中間データ・帳票の出力先。**初回実行で自動作成されたIDを設定** | ★ |
   | `EXPENSE_SPREADSHEET_ID` | 経費申請アプリのスプレッドシートID（支払明細に経費を載せる場合） | 任意 |
   | `INVOICE_REGISTRATION_NO` | インボイス登録番号（請求書に印字） | 任意 |

> ソース（実績・経費／シフト）へは**読み取りしか行わない**設計。出力はすべて別ファイル。

## 2. 実行順序

### 2-1. タブ確認（Step 1 相当）
```
runStep1()
```
- ログに全タブ名・実績行数・コース数が出る
- 実績タブが自動判定できなければ `JISSEKI_TAB_NAME` を設定
- 初回は `OUTPUT_SPREADSHEET_ID 未設定のため新規作成しました: <ID>` が出るので、
  そのIDをスクリプトプロパティに設定

### 2-2. M4 マスタ出力（Step 2 相当）
```
writeM4ToOutput()
```
- 出力ファイルに「M4_コース対応マスタ」タブが生成される
- ステータス「要確認」の行（複数エリア混在3件）は暫定=主エリア単価。運用で気になれば修正

### 2-3. 実績照合（Step 3）
```
writeReconcileReport('2026/02', '2026/02')
```
- 第1引数 = シフトのタブ名（例 `2026/02`。タブ名が違う場合は実際の名前を渡す）
- 出力ファイルに「照合_202602」タブ。ログに P1〜P4 件数
- **P1（予定あり実績なし）・P2（実績あり予定なし）・P3（案件相違）** を目視確認し、
  実績・シフトの入力漏れを是正（既存シートの修正はユーザー側で）

### 2-4. 請求・支払計算（Step 4）
```
runStep4('2026/02')
```
- 「請求書_202602_フェイト」「請求書_202602_k-dash」「支払明細_202602_<ドライバー>」
  タブが生成される

### 2-5. 検算（このStepの本丸）
出力された請求書が確定済み試算と一致することを確認:

| 請求先 | 小計(税抜) | 消費税(四捨五入) | 税込合計 |
|---|---|---|---|
| フェイト | ¥3,354,235 | ¥335,424 | **¥3,689,659** |
| k-dash | ¥1,187,809 | ¥118,781 | **¥1,306,590** |

- 一致 → 本稼働OK。月次運用は 2-3 → 2-4 の2コマンドのみ
- 不一致 → 照合レポートと「⚠単価未解決」注記行から差分行を特定

## 3. 月次運用（本稼働後）

毎月初に前月分を実行:
```
writeReconcileReport('<シフトタブ名>', 'YYYY/MM')   // 照合レポート確認
runStep4('YYYY/MM')                                  // 請求書・支払明細生成
```
PDF化は出力ファイルの各タブを「ファイル > ダウンロード > PDF」または既存の
軽貨物実績確認用GASのPDF出力の仕組みを流用（引き継ぎ資料 Step 4 参照）。
