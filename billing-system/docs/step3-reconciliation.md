# Step 3: 実績照合（フェーズ1）— 実装

作成日: 2026/07/19
前提: `docs/step2-m4-master.md`（M4確定版）／要件定義書 v5 F-001・F-002・フェーズ1

---

## 目的

シフト（予定）× 配送実績（実際）を **日付 × ドライバー × 案件/コース** で突き合わせ、
ヌケモレを検出する。一致した行のみを「確定実績」として Step 4（請求・支払）へ引き渡す。

| 判定 | 意味 | 扱い |
|---|---|---|
| **P1** | 予定あり・実績なし | 欠勤/未報告の疑い → 要確認 |
| **P2** | 実績あり・予定なし | 予定外稼働 → 要確認 |
| **P3** | 案件相違 | 予定と実績でエリア/案件が違う → 要確認 |
| **P4** | 一致 | ✅ 確定実績（Step 4へ） |

## 実装（`apps-script/Reconcile.gs`）

主な関数:
- `buildNameMaster_()` … M5名寄せ。ドライバー一覧タブの「氏名(姓)」「フルネーム」から
  姓⇔フルネームの解決表を生成
- `readShiftMonth_(tabName, nameMaster)` … シフトのドライバー×日付マトリクスを
  `{date, driver(フルネーム), cell, 分類}` に展開（稼働セルのみ）
- `classifyShiftCell_(raw)` … シフトセルを 宅配（エリア）/企業配（案件）/非稼働 に分類
- `reconcileMonth(shiftTab, 'YYYY/MM')` … 照合本体（P1〜P4判定）
- `writeReconcileReport(shiftTab, 'YYYY/MM')` … 結果を別ファイルの「照合_YYYYMM」タブへ出力

実行例: `writeReconcileReport('2026/02', '2026/02')`（第1引数=シフトのタブ名）

## 実データを踏まえた設計判断

### 1. 名寄せ（M5）が必須だった
引き継ぎ資料は「実績・シフトともフルネームで一致」としていたが、シフトの
**2026/03タブはドライバーが姓のみ**（赤羽・根本・山中…）。実績はフルネーム（赤羽眞…）。
→ ドライバー一覧タブの姓⇔フルネーム表で解決。8月タブ等フルネームの月はそのまま通る。

### 2. シフトセルは宅配・企業配・非稼働が混在
セル分類ルール（`classifyShiftCell_`、実セルで検証済み）:

| 入力例 | 分類 | 結果 |
|---|---|---|
| `上高井戸1大` / `上高井戸1小＋S` | 宅配 | 上高 |
| `中央` / `本町` / `中野S中央40以降` | 宅配 | 中野 |
| `キャピタル` / `ツイン東新橋2` / `南青山2` / `青山2` | 宅配 | 青山系 |
| `豊洲（体験）` / `企業配①:豊洲` | 企業配 | 豊洲 |
| `日比谷 Put` / `東武池袋` / `縁 ちよだ` / `建築現場` / `三茶` | 企業配 | 案件名 |
| `休` / `S待機` / `MTG` / `出張` / `企業配研修` / `○` / `-` | 非稼働 | 除外 |

- 注記（`＋S`、`（研修）`、`17時まで`、セル内改行）は `normalizeShiftCell_` で除去
- 具体コース/案件が取れず研修・待機語だけのセルは非稼働扱い（照合ノイズを削減）

### 3. 企業配 vs 宅配のズレ
実績シート（宅配数）には企業配の稼働は数量として出ない。シフトが企業配のみ・実績が
宅配のみの日は P3（案件相違: 予定=企業配/実績=宅配）として明示する。企業配の実績・
請求は Step 4 で企業配単価マスタ（付録A）側から別途集計する。

### 4. 仲山さんのコース空白行
実績のコース空白行は取引先でエリア相当を代替（`(取引先:フェイト)` 等）し、
シフト側の宅配エリアと厳密一致はしないため P2/P3 に寄りやすい。レポート上で可視化し、
Step 4の単価は確定ルール（フェイト¥167 / k-dash¥163）で計算する。

## 制約・検算方針

⚠️ 本エンジンは **GAS上で実シート（clean な getValues）** を読む前提で実装。
Drive連携の全タブ連結テキストはセル内改行で列崩れするため、**本番照合はGASで実行**する。
セル分類ロジックは実セル値で単体検証済み。月次の突き合わせ精度は
**Step 5（2026年2月分の検算）** でユーザー手元数字と突合して確定する。
