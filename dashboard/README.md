# Gooner運送事業部 経営分析・黒字化ダッシュボード

Gooner運送事業部の財務改善と「月利100万円の黒字化」の迅速な達成を目的とした、
インタラクティブ経営分析ダッシュボードです。要件定義は本リポジトリの
`docs/gooner-dashboard-requirements.md` を参照してください。

## セットアップ

```bash
npm install
npm run dev      # 開発サーバー
npm run build    # 本番ビルド（dist/ に出力）
npm run preview  # ビルド結果のプレビュー
npm run lint     # oxlint
```

## 主な機能

- **What-If シミュレーター**（画面左）: ヤマト稼働人数・配完数・企業配手数料率・
  管理者人件費削減率・車両固定費削減額の5つのスライダーで、月間最終経常損益を
  リアルタイムに再計算（`useMemo` によるキャッシュ）。黒字化目標（月利100万円）
  との GAP、達成に必要なヤマト稼働人数も自動算出。
- **4つの分析タブ**（画面右）: 全体PL／ヤマト宅配／企業配・スポット／現場課題の
  各ビューで、KPIサマリー・ドライバー個別貢献度・案件別中抜き率・現場バグトラッカー
  を表示。
- **エクスポート**: シミュレーション結果をテキスト整形してクリップボードにコピー
  （LINE・スプレッドシートへの貼り付け用）。
- **トースト通知**: `alert()`/`confirm()` は使用せず、iframe環境でも動作する
  独自のトースト通知UIを使用。

## データ

`src/data/` にモックのマスターデータ（ドライバー・固定諸経費・企業配案件・現場
バグログ）を定義しています。実運用では外部スプレッドシート／APIとの連携に置き
換えてください。計算ロジックの規定値は `src/data/constants.ts` に集約しています。

## 技術スタック

React 19 + TypeScript + Vite + Tailwind CSS v4 + lucide-react（アイコンはバンドル済みでオフライン・iframe環境でも動作）。
