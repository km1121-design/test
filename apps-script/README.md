# ダッシュボード連携用 Apps Script

配送実績ログ（Google スプレッドシート）を非公開のまま、ダッシュボードに必要な
集計値だけをJSONで返すためのGoogle Apps Scriptです。

## なぜ生シートを直接公開しないか

ダッシュボードは静的サイト（GitHub Pages）としてブラウザから直接データを取得する
構成のため、もし生のスプレッドシートを「リンクを知っている全員が閲覧可」にすると、
ドライバー個人の日次稼働実績がまるごと誰でも取得できる状態になってしまいます。
これを避けるため、間に「集計だけを返す」Apps Script APIを挟みます。

- 生シート（配送実績ログ）: 非公開のまま
- 支払管理表（銀行口座情報を含む）: **連携対象から完全除外**
- 公開されるのは「氏名 × コースの宅配完了数・ネコポス数」の集計のみ
  （単価・給与額そのものはAPIに含まれず、ダッシュボード側のコードで計算する）

## セットアップ手順

1. 配送実績ログのスプレッドシートを開く
2. `拡張機能 > Apps Script` を開く
3. `dashboard-api.gs` の内容をエディタに貼り付ける
4. `SHEET_NAME` を実際のシート（タブ）名に書き換える
5. `デプロイ > 新しいデプロイ` を選択
   - 種類: **ウェブアプリ**
   - 実行するユーザー: **自分**
   - アクセスできるユーザー: **全員**
6. デプロイして発行された `.../exec` で終わるURLを控える
7. そのURLを `dashboard/src/data/liveConfig.ts` の `LIVE_DATA_ENDPOINT` に設定する
   （このリポジトリを編集できる人に共有してください）

## データを更新したいとき

スプレッドシートの中身が更新されれば、ダッシュボードを開くたびに最新の集計が
反映されます（Apps Script側の再デプロイは不要）。ただし以下を変更した場合は
再デプロイが必要です。

- 列構成（列名）を変更したとき
- シート（タブ）名を変更したとき
- 集計ロジック自体を変更したとき

## 動作確認

デプロイ後、発行されたURLをブラウザで直接開いて、以下のようなJSONが返る
ことを確認してください。

```json
{
  "generatedAt": "2026-07-03T12:00:00.000Z",
  "periodFrom": "2026-06-17",
  "periodTo": "2026-07-03",
  "rows": [
    { "name": "山中一輝", "course": "中野本町1", "partner": "フェイト", "totalCompleted": 128, "totalNekopos": 60, "reportDays": 1 }
  ]
}
```
