# BAR 業務日報・売上管理システム

日々の売上報告・勤怠管理・目標進捗を一元管理し、手計算のミスをなくして「正しい
評価」と「利益の見える化」を行うためのダッシュボードです。要件定義は本リポジトリの
`docs/bar-dashboard-requirements.md` を参照してください。

## セットアップ

```bash
npm install
npm run dev      # 開発サーバー
npm run build    # 本番ビルド（dist/ に出力）
npm run preview  # ビルド結果のプレビュー
npm run lint     # oxlint
```

## 主な機能

- **日報入力**（LINE Bot送信の模擬）: 代表向け・スタッフ向けの入力フォームから
  送信すると、★自動計算項目（インセンティブ・当月累計・利益など）を含む
  「報告用」テキストを生成し、クリップボードにコピーできます（LINEグループへの
  転送を想定）。
- **サマリー**: 1部・2部それぞれの月次・年次の売上・経費・利益を一覧表示。
- **ダッシュボード**: 部門・月を切り替え、目標・売上・GAP・達成率・必要達成率・
  経費・人件費・スタッフ時給・インセン・来客数・客単価・利益を表示。
- **日次進捗管理**: 部門・月内の日ごとの売上・経費・利益・累計・GAP・達成率。
- **代表個人進捗管理** / **スタッフ個人進捗管理**: 個人ごとの月次KPIと日次実績。
- **マスター管理**: スタッフ、部門月間目標、インセン率・時給・税率のパラメータを
  編集可能。

## データ

`localStorage` にデータを保存するフロントエンド完結の実装です（LINE Bot・
サーバー側DBは未接続）。初回起動時は `src/data/seedData.ts` が当月1日〜本日分の
デモ用サンプル日報を自動生成します。ヘッダーの「デモデータをリセット」から
いつでも初期状態に戻せます。

計算ロジック（インセンティブ・利益・勤怠時間など）は `src/lib/calculations.ts` に
集約しています。実運用でLINE Messaging API Webhookやサーバー側DBに置き換える
場合も、このロジックはそのまま流用できます。

## 技術スタック

React 19 + TypeScript + Vite + Tailwind CSS v4 + lucide-react。`dashboard/`
（Gooner運送事業部ダッシュボード）と同一の技術スタック・ダークテーマの
デザイン方針を踏襲しています。
