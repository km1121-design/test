# 経費申請アプリ 仕様書

最終更新: 2026-07-14

## 1. 目的

経費の申請から確認・承認までをブラウザ上で完結させ、申請データを
**Google スプレッドシートにデータベース化**、**領収書画像を Google ドライブに保存**する。
これにより蓄積データを実績管理・分析ツールから読み込んで連携・分析できる状態にする。

## 2. システム構成

```
[利用者のブラウザ]                        [Google クラウド]
  経費申請アプリ (静的サイト)               ┌───────────────────────────┐
   - 画像OCR (Tesseract.js / 端末内)  ─┐   │ Apps Script Web App       │
   - 申請フォーム                      │   │  doPost: create/update/   │
   - 個人 / 管理者ダッシュボード        ├──▶│         delete            │
   - localStorage キャッシュ           │   │  doGet : 全件JSON          │
                                       │   └───────┬───────────┬───────┘
                                       │           ▼           ▼
                                       │   スプレッドシート   Google ドライブ
                                       │   (経費DB)          (領収書画像)
                                       │           │
[分析・実績管理ツール] ◀────────────────┘           │
   Looker Studio / BIツール / スプレッドシート集計 / JSON API / CSV
```

- フロントエンド: フレームワーク不要のバニラ JS（`expense-app/`）。GitHub Pages で公開。
- バックエンド: Google Apps Script Web App（`apps-script/Code.gs`）。サーバー・GCP設定不要。
- データストア:
  - **スプレッドシート**（`expenses` シート）= 経費データの正本（1申請1行）。
  - **Google ドライブ**（既定フォルダ「経費領収書」）= 領収書画像。行に URL を保持。
- 連携: 分析ツールはスプレッドシート／JSON API／CSV から同一データを参照。

### 2.1 保存モード

| モード | 条件 | 保存先 | 用途 |
| --- | --- | --- | --- |
| クラウド連携 | 設定でWebアプリURLを登録 | スプレッドシート＋ドライブ（正本）、localStorageはキャッシュ | 複数端末・分析連携 |
| ローカル | 未設定 | localStorage のみ | 試用・単一端末 |

クラウド連携時、通信失敗した申請は端末内の再送信キューに退避し、「再同期」で
スプレッドシートへ反映する（データ欠損を防止）。

## 3. 機能要件

### 3.1 画像解析（レシートOCR）
- レシート／領収書画像をアップロード（クリック選択・ドラッグ＆ドロップ）。
- Tesseract.js による**端末内OCR**。OCR自体は外部送信しない。
- 抽出項目: 金額（「合計」等を優先）、日付（和暦記号/西暦の各形式）、支払先（店名）。
- 抽出結果はフォームへ自動入力され、手修正可能。全文テキストも確認可能。

### 3.2 経費申請
- 入力: 日付（必須）、科目（必須）、支払先、金額（必須・1円以上）、摘要。
- 科目: 交通費／交際費／会議費／消耗品費／通信費／宿泊費／その他。
- 申請時: クラウド連携ONなら画像を Drive に、内容を Sheet に保存。OFFなら localStorage。
- 申請者は画面右上の「ログインユーザー（氏名）」で識別。

### 3.3 個人確認用ダッシュボード
- ログインユーザー自身の申請のみ表示。
- 集計: 申請件数、申請中件数、承認済み金額、申請中金額。
- 一覧: 日付・科目・支払先・金額・状態・領収書リンク・備考（却下理由）。
- 申請中のものは取消可能。

### 3.4 管理者確認用ダッシュボード
- 「管理者モード」ON で表示。全申請を対象。
- 集計: 総申請件数、承認待ち件数・金額、承認済み金額、科目別承認済み金額グラフ。
- 一覧: 申請者・日付・科目・支払先・金額・領収書リンク・状態・操作。
- 操作: 承認 / 却下（理由入力）/ 差戻。クラウド連携時はスプレッドシートへ反映。
- 検索: 申請者・店名。状態フィルタ。**CSV書き出し**で分析ツール取り込み。

## 4. データモデル（`expenses` シート）

`id, createdAt, applicant, date, category, vendor, amount, description, status,
reviewedAt, reviewer, reviewComment, imageUrl, imageFileId`

- `status`: `pending`（申請中）/ `approved`（承認済み）/ `rejected`（却下）。
- 詳細は `apps-script/README.md` を参照。

## 5. API

| メソッド | 内容 |
| --- | --- |
| GET `?token=` | 全申請を JSON 返却（分析・ダッシュボード読込） |
| POST `create` | 申請作成（画像base64があればDrive保存） |
| POST `update` | 状態更新（承認・却下・差戻） |
| POST `delete` | 申請削除（画像もゴミ箱へ） |

POST は CORS プリフライト回避のため `Content-Type: text/plain` で JSON を送信。

## 6. セキュリティ / 制約

- 認証は簡易（氏名入力＋管理者トグル＋任意の共有トークン）。本格運用では
  Web アプリのアクセス範囲を組織内に限定し、`SHARED_TOKEN` を設定する。
- OCR は端末内処理だが、申請時に画像・内容は自組織の Google 環境へ送信される。
- ローカルモードのデータは端末・ブラウザ間で共有されない。

## 7. 今後の拡張候補
- 承認ワークフロー（多段承認・部門別ルーティング）。
- 会計ソフト（freee / MF など）連携、月次締め・支払一覧の自動生成。
- Google OAuth による正式な個人認証と権限管理。
