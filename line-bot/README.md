# BAR業務日報 LINE Bot

`bar-dashboard/`（フロントエンドのデモ・可視化ダッシュボード）に対する、実際にLINEで
日報をやり取りするためのバックエンドです。要件定義は
`docs/bar-dashboard-requirements.md` を参照してください。計算ロジック（インセン
ティブ・利益・勤怠時間など）は `bar-dashboard` と同じ内容を `src/lib/calculations.ts`
に移植しています。

## できること

- 代表・スタッフがLINEで日報を送信すると、内容をパースしてSQLiteに保存
- 送信のたびに★自動計算項目（当月累計・インセンティブ・利益など）を計算し、
  「報告用」テキストを返信
- `REPORT_GROUP_ID` を設定すると、完成した報告用テキストを指定の「日報グループ」
  へ自動転送
- 同一日付・同一報告者の日報を再送すると、内容を上書き（最新の送信が正）
- スタッフ・部門目標・インセン率などのマスターデータはSQLiteで管理

LIFF（LINEアプリ内ブラウザで動くフォーム）は実装していません。日報は「ラベル: 値」
形式のテキストメッセージとしてBotに送信する方式です（後述のテンプレ機能で入力の
手間を減らしています）。

## セットアップ

### 1. 依存関係のインストール

```bash
cd line-bot
npm install
```

### 2. LINE Developersでチャネルを作成する（未作成の場合）

1. [LINE Developers Console](https://developers.line.biz/console/) にログインし、
   プロバイダーを作成（または既存のものを選択）する。
2. 「新規チャネル作成」から **Messaging API** チャネルを作成する。
3. チャネル作成後の「Messaging API設定」タブで以下を控える。
   - **チャネルシークレット**（「チャネル基本設定」タブ）
   - **チャネルアクセストークン（長期）**（「Messaging API設定」タブの下部で発行）
4. 「Messaging API設定」タブで以下を設定する。
   - Webhookの利用: オン
   - Webhook URL: `https://<デプロイ先のドメイン>/webhook`（後述のデプロイ後に設定）
   - 応答メッセージ: オフ（Bot からの応答と重複させないため）
   - あいさつメッセージ: 任意
5. Botを日報グループ、または代表・スタッフとの個人チャットに招待/友だち追加する。
6. 日報を集約する「日報グループ」を作る場合は、そのグループにもBotを招待し、
   グループIDを控える（`REPORT_GROUP_ID` に設定。取得方法は最下部のFAQを参照）。

### 3. 環境変数の設定

```bash
cp .env.example .env
```

`.env` を開き、`LINE_CHANNEL_ACCESS_TOKEN` / `LINE_CHANNEL_SECRET` を設定する。
`REPORT_GROUP_ID` は日報グループへの自動転送を使う場合のみ設定する（未設定の場合は
日報を送信したトークにそのまま返信するだけになる）。

### 4. マスターデータの投入

```bash
npm run seed
```

スタッフ（代表2名＋スタッフ6名の初期データ）、パラメータ（インセン率・時給・税率）、
直近3ヶ月分の部門目標をSQLiteに投入する。氏名や目標値は `src/db/seed.ts` を編集して
実態に合わせる（一覧編集用のマスター管理画面は未実装。将来的には
`bar-dashboard` のマスター管理画面をこのDBに接続する形に統合する想定）。

### 5. 起動

```bash
npm run dev     # 開発用（ファイル変更を監視）
npm run build   # 本番ビルド（dist/ に出力）
npm start       # ビルド済みのdist/index.jsを起動
```

Webhook URLとして外部からアクセスできるHTTPSエンドポイントが必要。ローカル開発時は
[ngrok](https://ngrok.com/) 等でトンネルを張り、`https://xxxx.ngrok-free.app/webhook`
をLINE Developersコンソールの Webhook URL に設定する。

### 6. LINE無しでロジックを確認する

実際のLINEチャネルを用意する前に、パーサー・計算・報告用テキスト生成だけを
ローカルで確認できる。

```bash
npm run cli -- - <<'EOF'
代表日報
日付: 2026-07-03
報告者: 竹原
通常売上: 120000
ハイエース売上: 30000
現金売上: 150000
クレカ売上: 150000
電子マネー売上: 50000
QR決済売上: 50000
EOF
```

## メッセージフォーマット

Botに以下のコマンドを送信できる。

- `テンプレ 代表` … 代表日報の記入用フォーマットを返信
- `テンプレ スタッフ` … スタッフ日報の記入用フォーマットを返信
- `ヘルプ` … 使い方を返信

テンプレをコピーし、`:` の後ろの値を書き換えて送信すると日報として記録される
（1行目の `代表日報` / `スタッフ日報` は書き換えない）。

```
代表日報
日付: 2026-07-03
報告者: 竹原
通常売上: 120000
ハイエース売上: 30000
個人既存来客数: 5
個人新規来客数: 2
クロスセル件数: 1
現金売上: 150000
クレカ売上: 150000
電子マネー売上: 50000
QR決済売上: 50000
スタッフ売上: 240000
イベント売上: 60000
本部売上: 100000
全体既存来客数: 20
全体新規来客数: 5
総評: 順調でした
当日予定: 予約2件
伝票: invoice_0703.jpg
経費1: 消耗品費/5000/文房具
経費2: 光熱費/8000/電気代
```

```
スタッフ日報
日付: 2026-07-03
氏名: 佐藤
勤務開始: 18:00
勤務終了: 24:00
休憩時間: 60
当日売上: 45000
売上詳細: ドリンク中心
既存来客数: 4
新規来客数: 1
クロスセル件数: 0
良い点: 常連対応が良かった
改善点: 在庫確認遅れ
```

補足:

- 日付は `YYYY-MM-DD`、勤務時刻は `HH:mm`（日またぎは24:00以降、例: 翌1時なら
  `25:00` のように24時間を超える表記も可）。
- 部門は報告者のマスター登録部門が自動で使われる。行を追加して
  `部門: 1部` のように上書きも可能。
- 報告者名・氏名は `npm run seed` で登録した氏名と完全一致させる。
- 経費は `経費1` `経費2` … と行を増やすと複数登録できる。

## デプロイ

Node.js を実行できる任意のホスティング先にデプロイできる（`npm run build` →
`npm start`）。代表的な選択肢:

- **Render / Railway / Fly.io**: リポジトリ連携で `line-bot/` をルートに指定し、
  Build Command に `npm install && npm run build`、Start Command に `npm start`
  を設定する。SQLiteファイルを永続化するため、永続ディスク（Volume）をマウントし
  `DB_PATH` をそのパス配下に設定する（永続ディスクがないと再デプロイ時にデータが
  消える点に注意）。
- **自前VPS**: `pm2` 等のプロセスマネージャで `npm start` を常駐させ、Nginx等で
  リバースプロキシしてHTTPS化する。

## データ

`better-sqlite3` を使ったファイルベースのSQLiteです（`DB_PATH` で保存先を指定、
既定は `./data/bar-line-bot.sqlite3`）。リポジトリ層（`src/db/repository.ts`）を
薄く分離しているため、将来Postgres等の外部DBに差し替える場合もこの層だけを
書き換えれば良い構成にしています。

## FAQ

**Q. グループトークのIDはどう調べる？**
A. Botをグループに招待した状態でグループ内から何かメッセージを送信すると、
Webhookイベントの `source.groupId` に含まれる。開発中は `npm run dev` のログに
出力されるように、必要であれば `src/index.ts` の `resolveSourceId` 呼び出し箇所に
一時的に `console.log` を追加して確認するとよい。

**Q. `bar-dashboard`（ダッシュボード）とデータは連携される？**
A. 現時点では連携していません。`bar-dashboard` はブラウザの `localStorage` に
保存するスタンドアロンのデモ・可視化フロントエンドです。将来的にこのLINE Bot
サーバーにダッシュボード用のAPIを追加し、`bar-dashboard`側の永続化層を
`localStorage` からAPI呼び出しに差し替えることで統合できます。
